---
name: Python style
---

- Follow PEP 8 and the repository's existing style.
- Prefer explicit typing where the surrounding codebase uses it.
- Keep functions small and focused.
- Do not introduce unnecessary abstractions.
- Prefer standard library utilities and existing project helpers before adding new code paths.
- Preserve existing logging patterns and configuration style.
- Preserve exception-handling patterns unless a change is explicitly requested.
